Grid Elements: Container










.container{
    width: {breakpoint max width};
    padding-left: 15px;
    padding-right: 15px;
}

The container has the purpose to provide padding so the content doesn’t touch the edge of the browser.

This is 15px ($gutter/2) on each side as seen in pink in the image.

You never need another container inside of a container.

Grid Elements: Row










.row {
    margin-left: -15px;
    margin-right: -15px;
}

The row provides the columns a place to live.

Rows also have a unique aspect of having 15px of negative margin on each side. The row would normally be constrained inside of the container's padding. The 15px negative margins push the row out over top of the containers 15px padding, essentially negating it. Why? Because of how the columns work.

Never use a row outside a container, it won’t work.

Grid Elements: Columns
















.col-md-3 {
    float: left;
    width: 25%;
}

.col-md-9 {
    float: left;
    width: 75%;
}

The columns now have 15px padding, seen in yellow. This padding means that the columns actually touch the edge of the row, which itself touches the edge of the container since the row has the negative margin, and the container has the positive padding.

But, the padding on the column pushes anything inside the column in to where it needs to be, and also provides the 30px gutter between columns.

Never use a column outside of a row, it won’t work.

Nesting

You can make new grids inside of a column. You add rows right inside the outer columns

you do not need to use them in containers.

The trick here is that the column acts the same way as a container:
it has 15px of padding that allows the row’s negative 15px of margin to overlay it.
